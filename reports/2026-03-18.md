# OpenClaw Security Report — 2026-03-18

## Verdict: FAIL

## Summary
The security scan results reveal multiple high and critical severity vulnerabilities in several dependencies, including @hono/node-server and fast-xml-parser. These vulnerabilities can lead to authorization bypass, cross-site scripting, and denial of service attacks. Additionally, there are misconfigurations in the Dockerfile that can lead to uncontrolled behavior and security issues.

## Critical Findings
- [HIGH] @hono/node-server via npm audit
  Issue: Authorization bypass for protected static paths via encoded slashes in Serve Static Middleware
  Fix: npm audit fix --force
- [HIGH] fast-xml-parser via npm audit
  Issue: Numeric entity expansion bypassing all entity expansion limits
  Fix: npm audit fix --force
- [CRITICAL] fast-xml-parser via trivy
  Issue: Cross-Site Scripting (XSS) due to improper DOCTYPE entity handling
  Fix: Update to version 5.3.5 or 4.5.4

## Warnings
- Misconfigurations in Dockerfile, such as missing HEALTHCHECK instruction and unspecified tag in 'FROM' statement

## Recommendation
Due to the presence of high and critical severity vulnerabilities with available fixes, it is strongly recommended to address these issues before deploying the code. Run 'npm audit fix --force' to update the vulnerable dependencies. Additionally, review and fix the misconfigurations in the Dockerfile to ensure the security and stability of the container images. It is also recommended to regularly monitor and update dependencies to prevent similar vulnerabilities in the future. The code should not be deployed until these critical issues are resolved.
