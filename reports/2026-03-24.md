# OpenClaw Security Report — 2026-03-24

## Verdict: FAIL

## Summary
The security scan results show multiple HIGH and CRITICAL severity vulnerabilities in various packages, including @angular/compiler, @angular/core, fast-xml-parser, jws, and node-forge. These vulnerabilities can lead to cross-site scripting, denial of service, and other security issues. Additionally, there are several misconfigurations in the Dockerfiles, including the lack of a HEALTHCHECK instruction and the use of an unspecified tag in the 'FROM' statement.

## Critical Findings
- [CRITICAL] fast-xml-parser via Trivy
  Issue: Cross-Site Scripting (XSS) due to improper DOCTYPE entity handling
  Fix: Update to version 5.3.5 or 4.5.4

## Warnings
- @angular/compiler: Angular: Cross-site scripting vulnerability in Template Compiler
- @angular/core: @angular/compiler: Angular has XSS in i18n attribute bindings
- fast-xml-parser: fast-xml-parser: Denial of Service via unlimited XML entity expansion

## Recommendation
Due to the presence of CRITICAL severity vulnerabilities with available fixes, it is recommended to update the affected packages to the latest versions before deploying the application. Additionally, the Dockerfiles should be updated to include a HEALTHCHECK instruction and a specific tag in the 'FROM' statement to ensure the security and stability of the application. It is also recommended to regularly scan the application for security vulnerabilities and misconfigurations to prevent similar issues in the future.
