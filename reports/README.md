# Automated Security Reports

This folder contains daily security summary reports generated by the GitHub Actions workflow.

## Format

Each report is a markdown file named `YYYY-MM-DD.md` with:

- **Verdict:** PASS / FAIL
- **Summary:** Overview of findings
- **Critical Findings:** High/critical severity issues with fix guidance
- **Warnings:** Lower severity or informational items
- **Recommendations:** Actionable next steps

## Workflow

Reports are automatically generated by `.github/workflows/security-pipeline.yml` which:

1. Runs **Trivy** to scan for CVEs, secrets, and misconfigurations
2. Runs **Hadolint** to check Dockerfile best practices
3. Parses results and writes the summary to this folder
4. Commits and pushes the report
