# Security Reports — Automated Vulnerability Scanning

Daily security scan reports for OpenClaw and related projects.

## 📋 What This Repo Does

Automated security scanning (via Trivy, Snyk, or similar) generates daily vulnerability reports. Each report analyzes:
- Dependency vulnerabilities (npm packages, system libraries)
- Dockerfile misconfigurations
- Critical security issues with fix recommendations

Reports are stored in `/reports/` as markdown files with ISO date names: `YYYY-MM-DD.md`.

## 📊 Report Structure

Each daily report includes:

| Section | Description |
|---------|-------------|
| **Verdict** | PASS / FAIL — overall security posture |
| **Summary** | High-level findings and fix availability |
| **Critical Findings** | CVE-level issues requiring immediate action |
| **Warnings** | Medium/low severity issues, config problems |
| **Recommendation** | Specific actions to resolve |

### Example: 2026-03-23 Report

```
Verdict: FAIL
Critical: fast-xml-parser (XSS) → update to 5.3.5
High: @angular/compiler vulnerabilities
Warnings: Dockerfile missing tags, HEALTHCHECK
```

## 🛠️ How Reports Are Generated

1. Security scan runs daily (likely via cron/GitHub Actions)
2. Results parsed into markdown
3. Committed to this repo with `security-bot` as author
4. Reports show PASS/FAIL status in commit messages

## 🔍 Using These Reports

### Quick Status Check
```bash
# Latest report
cat reports/$(date +%Y-%m-%d).md | grep Verdict
```

### Historical Trend
```bash
# Count PASS/FAIL over time
grep -r "Verdict:" reports/ | sort
```

### Action Items
Each report lists specific fixes. For critical issues:
- Update dependencies to recommended versions
- Add missing Dockerfile HEALTHCHECK
- Pin base images with tags (not `latest`)

## 🧩 Integration

This repo works with:
- **Trivy** (container scanning)
- **npm audit** / **yarn audit** (dependency scanning)
- **GitHub Actions** (automated runs)
- **OpenClaw** (consumes reports for security monitoring)

## 🚦 Security Policy

- Reports are read-only — don't edit manually
- Address CRITICAL findings within 24 hours
- HIGH findings within 72 hours
- Medium/Low: address during sprint planning
- PRs must not introduce new CRITICAL/HIGH findings
