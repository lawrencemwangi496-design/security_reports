# Security Reports Repository

Automated security scanning and analysis for the OpenClaw server environment.

## Overview

This repository contains two types of security reports:

1. **`reports/`** — Daily automated summary reports generated by GitHub Actions workflow
2. **`server_report/`** — Deep-dive analysis with raw scan outputs and detailed remediation steps

## How It Works

### GitHub Actions Workflow (`.github/workflows/security-pipeline.yml`)

Runs daily on schedule or manually triggered. Performs:

1. **Trivy scan** — Scans dependencies for CVEs and secrets
2. **Hadolint** — Checks Dockerfile best practices
3. **Generates summary** — Creates `reports/YYYY-MM-DD.md` with verdict and critical findings
4. **Commits and pushes** — Automatically updates the `reports/` folder

### Server Reports (`server_report/`)

Manual deep-dive analysis that complements the automated reports:

- **Raw scan outputs** — Full logs from Lynis, Rkhunter, ClamAV, Trivy
- **Threat analysis** — Severity, impact, and step-by-step fixes
- **System hardening** — Recommendations from Lynis audit

## Reports

### Automated Summary (`reports/`)

Quick daily status:
- **Verdict:** PASS / FAIL
- **Critical findings** with brief descriptions
- **Recommendations** for immediate action

### Detailed Analysis (`server_report/`)

Deep dive with:
- **Critical threats** — Full severity, component, fix commands
- **System hardening** — Kernel, SSH, filesystem recommendations
- **Rootkit investigation** — Suspicious file warnings
- **Malware scan** — ClamAV results

## Workflow Triggers

- **Schedule:** Daily at 05:00 UTC
- **Manual:** Via GitHub Actions `workflow_dispatch` with `scan_type` input
- **On push:** Not enabled by default
